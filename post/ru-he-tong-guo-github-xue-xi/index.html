如何通过GitHub学习？

Posted on 2020-02-03，2 min read

😀 目 的

当自己在学习新的知识时，为了能够提高学习效率，并且能够找到相对正确的学习路径，在GitHub上寻找资源是一个不错的选择。
可以通过GitHub 来搜索 Google 搜索不到的内容，大部分情况下我们想做的事情，已经有人已经做过。

😄 好 处

  1. 节省时间

    在 GitHub 上寻找相应的 Demo，了解原理、稍微尝试，再引入到项目中。
    尝试项目需要成本，如果内容较简单，可自己写Demo。
    例： python+demo

  2. 寻找脚手架：加快前期开发

    搜索技巧： 技术栈 + 技术名称， 寻找 awesome-xxx：可找到一些特定领域的资源
    是否采用脚手架，需要衡量上手的时间成本，是否比重头写快。
    想尽快的进入一个领域，不错的选择是通过awesome-xxx 寻找一些未知的可能性。
    例： python+awesome

  3. 模仿“轮子的轮子”

    一个完善的程序，会包含很多的细枝末节，这些内容对于上手是一个阻碍。学习框架直接阅读源码，容易陷入澡泽。
    通过造轮子的方式，能够掌握核心的部分，不妨先找一些小而精的轮子的轮子去造。 这种思路搜索Google 往往比 GitHub 搜索好用。
    例： xxx-like 或者 xxx-like framework 或者 仿Flask框架 或者 类flask。

  4. 学习资源

    GitHub 会有很多免费的资源、电子书、教程、甚至视频资料等等。
    例： 类型 + 笔记， python + 笔记， flask + 笔记， 书名， free-programming-books-zh_CN

  5. 数据及数据制作工具

    很多时候需要的数据，已经有人获取过了，可以在GitHub上直接拿来使用。比如一些爬虫工具，或者AI训练模型。
    例： face ai ， zhihu spider

😆 总 结

使用现成的工具来完成自己的工作，也是一种必备技能，并不是所有的技术都应亲自掌握。

人们开发软件的目的就是为了解决某些问题，如果这些问题已经被其他人解决过，为什么我们不去借用。

善用GitHub， 他山之石可以攻玉。
